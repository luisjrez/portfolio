---
title: How I work with AI
intro: AI isn't a buzzword on my résumé — it's part of how I ship. Here's how I fold it into real delivery.
---

- Spec-driven development :: I write a clear spec first, then drive implementation with Claude Code and clean architecture — turning intent into working, reviewed code instead of ad-hoc prompting.
- AI in the product, not just the process :: I've shipped user-facing AI features — voice-to-text with real-time streaming, plus title suggestions and content enhancement — so I understand latency, streaming, fallbacks and cost, not just the happy path.
- Prompts as versioned assets :: I manage production prompts in Langfuse instead of hardcoding them, so AI behaviour can be iterated and improved without a backend deploy.
- Multiple tools, right job :: Claude Code and Codex for building, Cursor for in-editor flow, Gemini and ChatGPT for exploration — I pick per task instead of forcing one tool.
- Review discipline :: AI accelerates the first draft; I still own architecture, edge cases and testing. The bar for what ships doesn't move.
